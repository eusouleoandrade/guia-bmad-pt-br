01 · Modelo mental

Decisões viram matéria.

Pessoa define intenção. Workflow conduz processo. Agente aplica especialidade. Artefato guarda decisão. Próxima etapa consome esse artefato.

O sistema

Não é modelo de IA. É método operacional.

BMAD é sistema modular de agentes, skills e workflows guiados. Não substitui IDE, gerenciador de projetos ou validação humana.

$bmad-help o que faço primeiro?
Regra de chamada

Clareza decide.

Objetivo claro: chame workflow. Problema confuso: converse com agente ou use help. Vários pontos de vista: party mode.

Agente

Quem pensa

Persona especializada: PM, UX, arquiteto, dev, analista ou tech writer.

Workflow

Como trabalha

Sequência disciplinada que cria, valida ou transforma entregáveis.

Artefato

O que permanece

Arquivo no disco. Sobrevive à troca de chat e alimenta próxima etapa.

Intenção+Especialidade+Processo=Artefato verificável
Instalação

Comece no projeto

Node.js 20.12+. Escolha módulo BMad Method.

npx bmad-method install

05 · Ritmo diário

Loop por story.

Depois de readiness e sprint planning, cada story atravessa cinco movimentos. Revisão pode gerar correção e nova rodada.

STORY
PRONTA
01Criar contexto
02Validar
03Implementar
04Revisar
05Corrigir

Contexto novo. Artefato contínuo.

$bmad-create-story prepara contexto. Ação validate verifica prontidão. $bmad-dev-story implementa e testa. $bmad-code-review revisa de forma adversarial independente.

Regra de ouro: nova janela para cada workflow. Não rode dois workflows que editam mesmos arquivos em paralelo. Commit depois de revisão e testes.
